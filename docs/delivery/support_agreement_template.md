## 1. Purpose

This document defines a simple support agreement template for freelance delivery.

The goal is to clarify what support includes, what support does not include, how long support lasts, and how future requests should be handled.

This protects both the freelancer and the client from unclear expectations after delivery.

## 4. Support Includes

Support may include:

- helping the client run the project
- explaining how to use the dashboard
- helping with CSV format questions
- checking simple CSV formatting problems
- fixing bugs related to delivered functionality
- correcting small text mistakes
- verifying installation steps
- explaining KPI meanings
- answering questions about the delivery documents

---

## 5. Support Does Not Include

Support does not include by default:

- adding new features
- adding new KPI calculations
- redesigning the UI
- adding authentication
- adding user roles
- adding database integration
- adding cloud deployment
- adding PDF export
- adding Excel export
- adding external file picker
- converting the project into a full ERP system
- rebuilding the project for another platform
- unlimited data cleaning
- unlimited client training
- long-term maintenance unless agreed separately

These requests should be treated as new paid work or a separate phase.

---

## 6. Bug Fixing Policy

A bug is a problem where the delivered project does not work as agreed.

Examples of bugs:

- dashboard fails to load with valid CSV files
- KPI calculation does not match the agreed logic
- report preview does not open
- configuration value does not appear correctly
- application crashes during normal use

Bug fixes related to agreed delivered features are included during the support period.

---

## 7. Data Issue Policy

The dashboard depends on correctly prepared CSV files.

If the CSV file has problems such as:

- missing required columns
- wrong column names
- invalid dates
- text inside numeric fields
- missing required values
- inconsistent product names
- incorrect category names

then the issue is considered a data preparation issue, not a software bug.

The freelancer may help explain the problem, but large data cleaning work is not included unless agreed separately.
